## When to use

To change the filename (slug) of a note that is wikilink-referenced elsewhere in the vault. The script atomically renames the file and rewrites every wikilink that referenced the old slug — including embeds, aliases, and heading anchors — while skipping code blocks and Foam's autogenerated link-reference region.

## When NOT to use

To move a note to a different folder without changing its slug: plain `git mv` is fine, wikilinks resolve slug-only.
To rename a script, template, or anything under `_meta/`: plain `git mv`.
To rename many notes at once: invoke this once per note, reviewing each dry-run. Do not wrap it in a loop without review.

## Invocation

```
_meta/features/rename-note/rename-note.sh <old-slug> <new-slug> [subfolder]
```

The script:
1. Checks the working tree is clean.
2. Locates the old file (searches vault or uses the subfolder hint).
3. Runs a dry-run and prints what would change.
4. Prompts `Proceed? [y/N]`.
5. On `y`: `git mv`, applies the sweep, `git add -A`, commits as `rename: <old> -> <new> (sweep wikilinks)`.

## Side effects

- Moves one file via `git mv`.
- Rewrites wikilinks in every `.md` under the vault (except protected paths).
- Creates one commit.

If anything goes wrong mid-flight, revert with `git reset --hard HEAD~1` (the commit is atomic).
